The Problem

Your external monitor deserves better

Most external monitors have a pixel density below 210 PPI. macOS refuses to offer HiDPI scaling on these panels, making text look noticeably softer than on your MacBook's Retina display.

Without RetinaScaler

  • Fuzzy text on external displays
  • Pixel-visible edges and aliased fonts
  • Only standard resolution modes available
  • Display settings scattered across System Settings

With RetinaScaler

  • Retina-quality text on all connected displays
  • Smooth, crisp 2x rendering on every element
  • Full range of HiDPI scaled resolutions
  • Unified menu bar panel for all monitors

How It Works

Two approaches, one goal

RetinaScaler uses two complementary methods to enable HiDPI. Choose the one that fits your workflow.

Recommended

HiDPI Native

Installs a display override that tells macOS your monitor supports HiDPI modes. Modes appear directly on the physical display at full refresh rate with zero overhead. One-time admin password, then it just works.

Full refresh rate Zero overhead Native macOS
Advanced

HiDPI Virtual Display

Creates a virtual display with higher backing resolution and mirrors your physical monitor to it. Enables any HiDPI resolution regardless of panel limitations. Best for resolutions that exceed native HiDPI capability.

Any resolution Virtual mirror Higher backing

Quick presets to position displays relative to each other: left, right, above, below.

Lives in your menu bar. Dark-mode card-based UI with per-display independent controls.

macOS blocks the app on first launch?

Since RetinaScaler is ad-hoc signed (not Apple-notarized), Gatekeeper will block it once. Right-click the app > Open, or go to System Settings > Privacy & Security > Open Anyway. This is only needed once.
